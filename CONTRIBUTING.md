# Contributing

Thanks for improving My Interview Vault. This project uses a docs-as-code
workflow: content changes happen through pull requests, and merged changes
publish to the live site.

## How to contribute

1. Fork the repository.
2. Create a branch with a short descriptive name.
3. Add or edit files in `docs/`.
4. Run `npm run build` to catch broken links and MDX errors.
5. Open a pull request.

## Writing guidelines

- Keep pages focused on one interview topic.
- Use practical examples and tradeoffs.
- Prefer clear explanations over long link dumps.
- Include complexity notes for coding problems.
- Add references when external facts or claims need a source.
- Keep headings descriptive so the generated sidebar and table of contents stay useful.

## Review expectations

Maintainers may ask for clarity, source links, formatting fixes, or narrower
scope before merging. A pull request is visible in GitHub immediately, but it is
not visible on the public site until it is merged and deployed.
