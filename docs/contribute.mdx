# Contribute

This project follows a docs-as-code workflow.

## Contribution flow

1. Fork the GitHub repository.
2. Create a branch for your change.
3. Edit or add a Markdown file under `docs/`.
4. Run the site locally and check formatting.
5. Open a pull request.

Your change is invisible on the live site until a maintainer reviews and merges
the pull request.

## Content guidelines

- Keep examples practical and interview-focused.
- Prefer explanations, tradeoffs, and common mistakes over long link lists.
- Add source links when a fact depends on a specific external reference.
- Use headings, bullets, and code blocks so pages are easy to scan.
- Keep each page focused on one topic.
