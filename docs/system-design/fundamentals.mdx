# System Design Fundamentals

System design interviews reward structured thinking more than memorized
architecture diagrams.

## Interview flow

1. Clarify requirements.
2. Estimate scale.
3. Define APIs and data models.
4. Sketch the high-level architecture.
5. Identify bottlenecks.
6. Discuss tradeoffs and failure modes.

## Core concepts

- Load balancing
- Caching
- Queues and asynchronous processing
- Database indexing
- Replication and sharding
- Observability

## Practice prompts

- Design a URL shortener.
- Design a notification system.
- Design a collaborative document editor.
- Design a feed ranking service.
