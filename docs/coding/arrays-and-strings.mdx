# Arrays and Strings

Arrays and strings show up in many first-round interviews because they test
indexing, invariants, and careful edge-case handling.

## Patterns

- Two pointers
- Sliding window
- Prefix sums
- Hash maps for frequency counts
- In-place partitioning

## Checklist

- What should happen for an empty input?
- Are duplicates meaningful?
- Can the input be modified?
- Is the target runtime linear or logarithmic?
- Does the solution need stable ordering?

## Practice prompts

- Find the longest substring without repeated characters.
- Merge intervals after sorting by start time.
- Return all pairs that sum to a target.
- Compress repeated characters in place.
