# Dynamic Programming

Dynamic programming is useful when a problem has overlapping subproblems and a
choice structure that can be expressed as a recurrence.

## How to approach

1. Define the state.
2. Write the recurrence.
3. Pick base cases.
4. Decide memoization or tabulation.
5. Check time and space complexity.

## Common shapes

- One-dimensional decisions, such as climbing stairs or house robber.
- Two-dimensional grid paths.
- Subsequence and substring comparisons.
- Knapsack-style include-or-skip choices.

## Interview explanation template

State what each cell or function call means before writing code. Then explain
why the recurrence covers every valid choice exactly once.
